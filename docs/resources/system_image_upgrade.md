# vyos_system_image_upgrade (Resource)
<center>

Operational
⯯
**Install a pinned VyOS ISO via /image and optionally reboot**

This resource is intentionally imperative (it performs actions). Best practice is to run it in a separate apply (e.g. `-target`) because rebooting will interrupt other resources.

</center>

## Schema

### Required

#### iso_url
- `iso_url` (String) HTTP(S) URL of the ISO to install (passed to the VyOS `/image` endpoint).
#### target_name
- `target_name` (String) The expected image name/version string to check for in `show system image` output (e.g. `1.5-rolling-YYYYMMDDHHMM`).

### Optional

#### backup_before
- `backup_before` (Boolean) Whether to call `/config-file` `save` before installing the image.
#### backup_config_file
- `backup_config_file` (String) Optional file argument for `/config-file` save. If empty, VyOS saves to `/config/config.boot`. You may also try an `scp://...` URL if your VyOS build supports it.
#### delete_on_destroy
- `delete_on_destroy` (Boolean) If true, deletes `target_name` via `/image` when this resource is destroyed.
#### reboot_after
- `reboot_after` (Boolean) Whether to call `/reboot` after the image install.
#### reboot_path
- `reboot_path` (List of String) Arguments passed as the `path` array to the `/reboot` endpoint. The docs use `[&#34;now&#34;]`.

### Read-Only

#### id
- `id` (String) The ID of this resource.
#### installed
- `installed` (Boolean) Whether the target image is present according to `/show` `system image`.
#### last_response
- `last_response` (String) Last response text from the API action (backup/install/delete).
#### show_output
- `show_output` (String) Raw output from `/show` `system image` (useful for debugging).
